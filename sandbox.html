Text styles

Text is a fundamental part of any web page. It is used to communicate information to users, provide instructions, and create a visual hierarchy on the page. The default text styles in Herbenspice are designed to be simple, clean, and easy to read. They are based on a modular scale and use a combination of font sizes, line heights, and letter spacings to create a harmonious typographic system.

CSS frameworks have often used various resets and normalizations to remove default browser styles and create a consistent starting point for styling text. Herbenspice builds upon the default styles provided by the browser and enhancing them with custom styles. This approach helps to create a more consistent experience across different browsers and devices while still allowing for customization and flexibility.

Body text

Size

The default body text style in Herbenspice is designed to be easy to read and comfortable for long-form content. From the CSS variable token --base-scale: 112.5% (18px for Normal text), everything is scaled to make the text larger and more readable without being too large or overwhelming. Relative size units rem and em ensure that text scales proportionally with the base font size. This makes it easy to adjust the size of text across the entire page by changing the base font size in the :root element.

There are utility class for setting the size of body text using T-shirt sizes from .size:xxs to .size:xxxxl. These classes can be applied to any element to set the text size. These are mainly for special display text or adjustments to body text, not headings.

.size:xxs (0.675rem) is the smallest allowed text size. It should rarely be used.

.size:xs (0.75rem) is the second smallest text size in Herbenspice. It is used for fine print, legal disclaimers, and other text that is not the primary focus of the page.

.size:s (0.875rem) is a small text size in Herbenspice. It is used for captions, footnotes, labels, and other text that is less important than the main content on the page.

.size:m (1rem) is the default text size in Herbenspice. It is used for body text, paragraphs, and other main content on the page.

.size:l (1.25rem) is a large body text size in Herbenspice. It is used for text that needs to stand out from the main content on the page.

.size:xl is 1.5rem

.size:xxl is 2.125rem

.size:xxxl is 3rem

.size:xxxxl is 4rem

Headings

Headings are used to create a hierarchy of content on a page. They should be used to organize content and make it easier for users to scan and understand the information on the page.

Default styling for Headings 1–6

The styles for headings apply to the HTML elements <h1> through <h6>. There are also explicit classes to create the default styling for each heading level, such as .hed-1 for <h1>. This is useful for cases where you need an arbitrary style for a heading or other text that does not semantically match the heading level.

Heading 1

This is the default styling applied to an <h1> element, of which there is always only one per HTML page. (To achieve this styling on any other element, apply the class .hed-1.)

Heading 2

This is the default styling applied to an <h2> element, which is used to create the main sections of a page or divisions of text content. (To achieve this styling on any other element, apply the class .hed-2.)

Heading 3

This is the default styling applied to an <h3> element, which is used to create subsections of a page or divisions of text content. This is often the lowest level of heading used in a page. (To achieve this styling on any other element, apply the class .hed-3.)

Heading 4

This is the default styling applied to an <h4> element, which is used to for further divisions of text content. (To achieve this styling on any other element, apply the class .hed-4.) This styling is often used for subdued divisions within running text that do not require a full section break.

Heading 5

This is the default styling applied to an <h5> element, which is used for minor divisions of text content. (To achieve this styling on any other element, apply the class .hed-5.) This styling might be used to define area groups of a UI container, for example.

Heading 6

This is the default styling applied to an <h6> element, which is used for the smallest divisions of text content. (To achieve this styling on any other element, apply the class .hed-6.)

Custom styling for headings

In addition to the default styling for headings, there are custom classes that can be applied to any element to create a unique heading style. These classes are useful for creating special headings that stand out from the rest of the content on a page.

Heading 0

The custom class .hed-0 is used to create a jumbo heading that is larger than the default <h1> element. This can be useful for creating a dominant hero section, such as on a Home page, or other prominent heading on a page. It can be applied to any element.

Herbenspice

Herbenspice is a semantic design system for vanilla front-end code built by a UI/UX designer. It aims to bootstrap lightweight web projects, documentation environments, and rapid prototypes with minimal tooling, leveraging the power of the browser with opinionated design sense and built-in accessibility.

Semantic paradigm

Imperative utility-first CSS frameworks like Tailwind CSS are popular with developers working without a designer or the constraints of consistency, but they can lead to bloated, unmaintainable code and a lack of semantic meaning.

Lightweight with no build tools

Intended for vanilla HTML, CSS, and JavaScript, Herbenspice uses CSS variables for design tokens, requiring no build tools. It's lightweight and easy to integrate into existing projects, no matter what tools or frameworks you're using.

Accessibility-first

Herbenspice offers a solid foundation for accessible design. It uses semantic HTML elements and ARIA attributes to ensure that all users can access and interact with your content.



Herbenspice is the personal design system and portfolio project of Michael Crowe, a digital product manager with a background in technical documentation, UI/UX design, and front-end development.
